AI履歴統合コマンド

目的
- 既存の分断された `ai_conversations` レコードを安全に統合する。dry-run オプションで影響を確認できます。

使い方
- 変更を本番反映する前に必ずバックアップを取ってください（DBダンプ）。

1) dry-run（影響確認）

    php artisan ai:consolidate-history --dry-run --minutes=30

  - `--minutes` は同一グループ内での時間差の閾値（分）です。デフォルトは 30 分です。

2) 実行（統合）

    php artisan ai:consolidate-history --minutes=30

注意点
- 統合ルール（現実装）: `ai_conversations` を `user_id` と `title` の先頭20文字のプレフィクスでグループ化し、同グループ内で作成時刻が閾値（デフォルト30分）以内のものを、最も早い会話にまとめます。まとめた会話のメッセージは早い会話へ移動され、後発の会話レコードは削除されます。
- 既存の履歴を厳密に統合したい場合は、閾値やプレフィクス長を調整してください。必要ならより複雑なアルゴリズム（類似度やメタ比較）に拡張できます。

運用
- コスト管理: 管理画面で `max_tokens` と `temperature` を厳格に設定し、`reverb.ai_max_tokens_hardcap` を設定しておくことを推奨します。
- 追加の安全策: 各リクエストに対するトークン推定と警告、管理者向けのクォータ監視アラートを追加可能です。
