# Toast（トースト）ガイドライン

目的

- アプリ全体で一貫したトースト通知（短時間のフィードバック）を提供するための設計と実装ルールを定めます。

対象読者

- フロントエンド開発者、QA、デザイナー

チェックリスト（最低限守ること）

- [ ] 共通のトースト API を使う（例: `useToasts().showToast(...)`）
- [ ] トーストの位置・アニメーションは共通コンポーネントで制御する
- [ ] 成功/失敗/警告/情報 の4種類のバリアントを用意する
- [ ] アクション（Undo 等）はオプションとしてサポートし、handler は id を受け取る（可能なら副作用をキャンセルできる）
- [ ] 画面リーダーに届くようアクセシビリティ属性を付与する

設計方針

- 短時間で消える一時的メッセージ。重要な情報はモーダルやページ内の永続的 UI にする。
- トーストは副作用の結果（保存成功、保存失敗、削除など）に利用する。ユーザー操作の必須確認には使わない。
- トーストは視認性を優先するが、画面の主要 UI を邪魔しない位置（右下が推奨）に固定する。

API 仕様（推奨）

- Composable: `useToasts()` を利用する（共有インスタンス）

    const { toasts, showToast, dismissToast } = useToasts();

- showToast signature (推奨)

    showToast(message: string, type: 'info'|'success'|'error'|'warning' = 'info', duration = 3000, action = null) => id
    - message: 表示テキスト（短く）
    - type: 表示スタイル
    - duration: ミリ秒。`0` または `null` で自動消去しない
    - action: { label: string, handler: Function }（任意）
    - 戻り値 id を返し、`dismissToast(id)` で明示的に閉じられる

実装パターン

- 共通の `useToasts` をシングルトンとして提供し、`ToastContainer` を `AppLayout` に置いて全ページで表示する。
- コンポーネント側は `showToast(...)` を呼び出すだけにする。
- Undo の場合は、action.handler 内でサーバー呼び出しのキャンセルやローカルのロールバック処理を行う。

UI / スタイル

- 位置: 右下固定（`fixed bottom-4 right-4`）
- サイズ: 最大幅 320px 程度（`max-w-sm`）
- 色: 成功=緑, エラー=赤, 警告=黄, 情報=灰
- テキスト: 1行を推奨。長文は切り、必要なら詳細はモーダルへ誘導
- アニメーション: フェード + 少しの translateY（0→8px）で入出力

アクセシビリティ

- role="status" を設定して、画面リーダーに非同期のフィードバックを伝える
- action ボタンはキーボードフォーカス可能にする（tabindex は不要）
- 重要な操作の確認はトーストではなく aria-live 領域やダイアログを使う

Undo / アクションボタンのガイドライン

- アクションは短時間で意味のあるものに限る（例: 直前の削除を取り消す）
- Handler は副作用を元に戻すロジックを持つ。可能であれば optimistic な UI を使い、Undo はサーバーへキャンセルリクエストを送る
- 例:

    showToast('項目を削除しました', 'success', 5000, {
    label: '取り消す',
    handler: () => undoDelete(itemId)
    });

テスト方針

- Unit: `useToasts` の show/dismiss の挙動（配列に追加される/削除される）をテスト
- Integration/E2E: ボタン操作でトーストが表示されること、Action ボタンが実行されることを確認
- Accessibility: スクリーンリーダーで role="status" が読み上げられるか（手動確認）

移行と整備計画（この WorkItem の後）

1. 既存ページのトースト呼び出しを `useToasts().showToast` に置き換える（PR 単位で段階的に）
2. `AppLayout` に `ToastContainer` を置き、全ページ共通にする（既に実装済みのはず）
3. デザイナーと協議してバリアントの色とフォントを最終決定する
4. 小さな E2E テスト群を追加（表示 + アクション検証）

QA チェックリスト（PR レビュー用）

- [ ] showToast が直接呼ばれていない（ユーティリティ経由で呼んでいる）
- [ ] 同一ページで複数トーストが重ならない／見切れない
- [ ] action がある場合は handler が実行されトーストが消える
- [ ] 画面リーダーで問題なく読み上げられる

推奨コード例（Edit.vue での使用）

// 保存成功
showToast('WorkItem を保存しました', 'success', 4000, {
label: '取り消す',
handler: () => {
// ここで undo ロジックを実行
// 例: API 呼び出しで削除を取り消す、もしくはローカルで状態を復元
}
});

運用メモ

- トーストは短命のフィードバック用。長時間残すべきメッセージはトースト以外の UI を検討する。
- 重大なエラーはトーストに加えログに記録し、必要ならアラート/ダッシュボードで追跡する。
